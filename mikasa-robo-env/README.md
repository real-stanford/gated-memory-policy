# MIKASA-Robo Env

Simulation side for evaluating Gated Memory Policies trained from [`imitation-learning-policies`](https://github.com/yihuai-gao/gated-memory-policy/tree/main/imitation-learning-policies) on MIKASA memory-intensive manipulation tasks.

## Quick Start

```bash
conda env create -f env.yml
conda activate mikasa
pip install -e . --no-deps
```

## Evaluation

### Local (single checkpoint)

Please change the env-id and the checkpoint path accordingly. Available env-ids/checkpoint-names are:
- ShellGameTouch-v0 / mikasa_shell_game_touch
- InterceptMedium-v0 / mikasa_intercept_medium
- RememberColor3-v0 / mikasa_remember_color_3
- RememberColor5-v0 / mikasa_remember_color_5
- RememberColor9-v0 / mikasa_remember_color_9


```bash
rm -r eval_results/ShellGameTouch-v0 # delete previous results. Skip if there are no previous results.
python eval/mikasa_eval.py \
    --env-id ShellGameTouch-v0 \
    --checkpoint ../imitation-learning-policies/data/checkpoints/mikasa/mikasa_shell_game_touch_diffusion_memory.ckpt \
    --num-envs 50 \
    --abs-joint-pos \
    --seed 42 # Use a different seed than training (seed 0)
```
> If you run into `OSError: Downloaded file's SHA-256 hash does not match record: ...`, please comment out the error line `"$CONDA_PREFIX/lib/python3.11/site-packages/mani_skill/utils/download_asset.py", line 127` replace with `pass`, delete `~/.maniskill` and run the script again.

> When using `seed=0` to eval, the success rate will be significantly higher. This is because the PPO oracle policy generates data on a batch of episodes with seed 0.

Rollout data will be saved in `eval_results/` directory.

Three control modes are available:

| Flag              | What the policy predicts | Execution                          |
| ----------------- | ------------------------ | ---------------------------------- |
| _(default)_       | 8D joint deltas          | `pd_joint_delta_pos` PD controller |
| `--abs-joint-pos` | Absolute target qpos     | `pd_joint_pos` PD controller       |
| `--direct-qpos`   | Absolute joint positions | Teleport (no physics)              |

Due to the high delta action noise generated by the PPO oracle policy, we train our model on `--abs-joint-pos` mode. `--direct-qpos` mode is not strictly a control mode, more a debugging step by asking the policy to predict the absolute joint positions.

### Distributed (multi-checkpoint sweep)

Three terminals coordinate via [robotmq](https://github.com/yihuai-gao/robot-message-queue/):


```bash
# 1. Policy server — loads checkpoints, serves inference (imitation-learning-policies env)
cd ../imitation-learning-policies
conda activate imitation
export PYTHONPATH=.:$PYTHONPATH
python scripts/run_policy_server.py
```

```bash
# 2. Env server — runs MIKASA sims on GPU, evaluates, reports results (mikasa env)
bash shell_scripts/serve_mikasa_env.sh 0 18765
```

```bash
# 3. Orchestrator — queues checkpoints, collects results (imitation-learning-policies env)
cd ../imitation-learning-policies
conda activate imitation
export PYTHONPATH=.:$PYTHONPATH
python scripts/serve_mikasa_checkpoints.py
```

Rollout data will be saved to `../imitation-learning-policies/eval_output/` directory.

> **Note:** The env server defaults to **GPU 1** and port `18765`. On a single-GPU machine, pass `0` explicitly: `bash shell_scripts/serve_mikasa_env.sh 0`. Override the policy server address with `--policy-server` (env server) or `--port` (orchestrator).

## Camera Resolution

Environments render at **128x128** (both third-person and wrist cameras). The policy expects **256x256**, so the eval pipeline bilinear-upsamples automatically before inference (`eval/policy.py`). 

All the training data is collected at 128x128 resolution. Therefore, keeping the rendering resolution at 128x128 is preferred.

If you want to have the policy working on higher resolution both during training and testing, please change the following:

- **Third-person** (`base_camera`): edit `CameraConfig` in each file under `mikasa_robo_suite/memory_envs/`
- **Wrist** (`hand_camera`): edit `panda_wristcam.py` in your `mani-skill` site-packages
